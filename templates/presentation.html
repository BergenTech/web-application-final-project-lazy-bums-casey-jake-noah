BT Club4Clubs

Casey Lee, Jake Park, Noah Kim

Overview

Our project was created to address the many current issues with how clubs are managed in BT including :

1. Difficult and obscure access to finding and joining clubs

2. No centralized hub of information where students can find all information relevant to clubs

3. Lack of organization of club events and leadership

Technology Used

Database Management - SQLite3

Backend - Python (Flask Framework)

Frontend - HTML/CSS/JavaScript, Bootstrap 5

Features - Student End

View clubs directory

Join / leave clubs

View admin approved events & calendar

Have a profile (major, year)

View announcements from club leaders / teachers

Features - Teacher / Club Leader End

Teachers must be promoted by admin to lead the club (automation once bergen.org emails approved)

Teachers can make announcements

Teachers can create events to admin for approval

Features - Admin End

Delete users

Delete clubs

Approve event

Promote users to leader

Challenges

Jinja errors (displaying recent messages, times, usernames, etc.)

Admin security errors

Teacher permission structure (Completely different from user and admin)

Solutions

Jinja Errors -> Fix: Use elaborate documentation to trace data flow and interaction between front-end templating and back-end data

Admin security errors -> Fix: Utilize a wrapper function to check if the logged in user possesses the privileges to view the admin page

Impact

Centralizes entire BT community in one app

Helps students explore interests in extracurricular activities

Establishes a welcoming environment for students confused about the current disorganized state of BT clubs
